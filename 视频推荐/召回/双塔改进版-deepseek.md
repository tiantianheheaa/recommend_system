要将双塔召回模型写出技术深度与亮点，最好的思路是**摆脱“纯点击率内积”的套路，引入长序列多兴趣建模、多目标优化与自适应负采样**，形成一套贴合短视频场景的复杂召回系统。下面是一个可直接用于简历的完整改进方案，覆盖 label 定义、结构设计、特征输入、正负样本选择，并突出“分位数召回”这一差异性创新。

## 方案总览：多兴趣长期序列建模 + 时长分位数多目标 + 混合对抗负采样的双塔召回模型

**核心亮点：**
- 用户塔采用 **长序列兴趣提取（SIM） + 多兴趣胶囊网络**，解决兴趣遗忘与单向量表达瓶颈  
- 引入 **播放时长分位数预估** 作为第二召回目标，与点击目标联合训练，召回高时长潜力内容  
- 设计 **混合自适应负采样**（in-batch 随机负 + 流行度纠偏负 + 在线硬负挖掘），显著提升模型判别力  
- 物品塔融合 **多模态预训练向量**，增强冷启动与语义匹配

## 1. Label 定义：从“点击”走向“深度消费”

| 目标 | Label 类型 | 定义与加工 | 使用场景 |
|------|------------|------------|----------|
| **主目标：有效点击** | 二分类 0/1 | 视频卡片曝光且 **播放时长 ≥ 3 秒**（或有点击行为），标记为正；仅曝光未点击为负 | 保证基础相关性 |
| **辅目标：时长分位数** | 回归（分位数标签） | 对正样本（播放>0），计算 `play_ratio = 播放时长 / 视频时长`，截断至 [0,1]；直接使用该值作为 label，训练 0.9 分位数预估 | 召回“高沉浸潜力”内容，替代传统的时长加权 |

**为什么用分位数而不是均值？**  
播放时长分布极端长尾，MSE 回归均值会被极少长视频主导；Pinball Loss 天然抗异常值，且 0.9 分位数对应的“你很可能看这么久”具有明确业务含义（乐观估计），便于定向高留存召回。

## 2. 双塔结构设计

```
用户塔：长期行为序列 → 多兴趣提取 → 多向量输出
物品塔：多模态特征融合 → 单向量（主塔）+ 辅助向量（分位数塔）
相似度：用户多向量与物品单向量逐一计算，取最大值作为最终得分
```

### 2.1 用户塔：长期序列 + 多兴趣解耦

**特征输入：**
- **短期行为序列**：近 50 条交互视频 ID（点击/完播/点赞），带时间差、行为类型
- **长期行为序列**：近 3 个月有效播放视频 ID，长度可达数千，通过 SIM 的 **两阶段检索**（hard search）筛选与候选视频类目/作者相近的 Top-K（如 K=200）参与注意力计算
- **用户属性**：年龄/性别/城市等级/设备，离散化后 embedding
- **实时上下文**：当前请求时间、网络环境、拉活间隔

**结构设计：**
1. 长期序列先用 SIM 的通用搜索单元（基于类目、作者等哈希桶）快速截断，然后用 Target Attention（以候选物品向量为 query）聚合长期兴趣向量。
2. 短期序列用 Transformer 或 GRU 提取时序向量。
3. 将长期向量、短期向量、用户属性 embedding 拼接，送入 **多兴趣动态路由胶囊网络**（MIND），输出 **M=3~5 个用户兴趣向量** \(u_1, u_2, ..., u_M\)。

**优势：** 一个用户向量无法同时表达“篮球、萌宠、美食”三种兴趣，多向量召回能大幅提升覆盖率与多样性。

### 2.2 物品塔：多模态 + 分位数头

**特征输入：**
- **视频 ID embedding** + **作者 ID embedding**（核心记忆特征）
- **视频多模态语义向量**：利用预训练模型（如 CLIP 提取视觉帧、BERT 提取标题/OCR文本），离线存为固定 256 维向量，或端上轻量 finetune
- **浅层统计特征**：视频时长（分段）、近 7 天 CTR/完播率/点赞率、发布时间（离散化）
- **内容标签**：类目、话题、bgm 曲风等

**结构设计：**
- 所有特征 concat → MLP（带残差）→ LayerNorm → 输出物品主向量 \(v_{click} \in R^d\)，用于点击目标。
- 分路设计：共享底层，额外接一个轻量 **分位数头**（如单层全连接），输出标量 \(q_{0.9}\)，经过 Sigmoid 映射到 (0,1)，预测该物品被用户观看的 0.9 播放比例分位数。  
  为保证向量检索效率，**分位数预估不参与多向量检索**，而是：
  - 在线召回时，仍用主向量 \(v_{click}\) 与用户多向量做点积，按点击分召回 Top-N。
  - 在粗排或精排阶段，可直接使用该塔输出的 \(q_{0.9}\) 分数作为排序特征；或在召回内部做 **双通道召回**：额外维护一个基于 \(q_{0.9}\) 的物品嵌入（将物品塔输出作为另一个向量），但工程复杂。简历中可描述为 **“联合训练下的时长分位数表征可内积，为轻量级高时长召回提供基础”**，展示前瞻设计。

**实际落地简化版：** 直接让物品塔输出 **两个向量**：\(v_{click}\) 和 \(v_{duration}\)。训练时，点击损失作用在 \(v_{click}\) 上，分位数损失作用在 \(v_{duration}\) 的内积上（物品塔输出 \(v_{dur}\)，用户塔对应输出一个专门的分位数兴趣向量）。这样可实现双目标可检索召回，是更强方案。

## 3. 模型训练：样本选择与损失函数

### 3.1 正负样本定义

| 样本类型 | 来源 | 作用 | 标签 |
|----------|------|------|------|
| **正样本** | 曝光且播放时长 ≥ 3s 的日志 | 点击目标的正例 | click_label=1, play_ratio 根据实际播放计算 |
| **负样本** | ① 随机负采样：全局均匀采样视频；<br>② 同 Batch 负采样：其他用户的正样本作当前用户的负样本（In-batch Negative）；<br>③ **硬负样本**：曝光但未点击 / 播放<3s 的样本，以及**在线挖掘的“高分未召”样本**（用近期高点击用户召回了但未曝光的视频，标记为 hard negative） | 提高模型分辨最难样本的能力 | click_label=0, play_ratio=0 |
| **时长分位数专用的正样本** | 所有产生播放的样本（包括播放<3s 但>0s），使用 play_ratio 作为回归目标 | 增加分位数标签的密度 | 仅用于分位数损失 |

**样本配比：** 每个正样本搭配 K 个负样本（如 1:4），其中 2 个随机负，1 个 batch 负，1 个硬负（通过采样池控制）。

### 3.2 损失函数

**总损失：**
\[
L = L_{click} + \alpha L_{quantile}
\]
- \(L_{click}\)：二分类交叉熵，作用于用户-物品对的内积 \(s_{click} = \max_{i\in\{1..M\}} (u_i \cdot v_{click})\)，经 sigmoid 后与点击标签计算。取 max 体现多兴趣中最佳匹配。
- \(L_{quantile}\)：Pinball Loss，作用于时长预估头输出 \(q\) 与真实 play_ratio \(y\)：
\[
\rho_{0.9}(y, q) = 
\begin{cases}
0.9 \cdot |y - q|, & y \ge q \\
0.1 \cdot |y - q|, & y < q
\end{cases}
\]
仅对正样本（有播放）计算该损失，避免大量 0 值样本淹没模型。若采用双向量检索版，则 \(q = \text{sigmoid}(u_{dur} \cdot v_{dur})\)。

**优化细节：**
- 使用混合精度训练与 L2 正则。
- 对多兴趣向量增加 **diversity loss**（余弦相似度惩罚），防止兴趣向量塌缩。
- 在分位数头上增加 Huber 平滑，提高早期训练稳定性。

## 4. 特征输入与 Embedding 工程（简历展开点）

| 类别 | 特征举例 | 处理方式 |
|------|----------|----------|
| 用户ID | user_id | 低维 embedding（如 dim=64） |
| 用户画像 | 年龄、性别、活跃度、城市等级 | 分桶离散化后 embedding |
| 用户实时 | 请求时间(小时/星期)、网络(WiFi/4G) | 离散 embedding 或 cyclic 编码 |
| 短期行为 | 近 50 条视频ID + 行为类型 + 停留时间 + 时间差 | Transformer 序列，position encoding 用时间差 |
| 长期行为 | 近 3 个月视频ID（千级） | SIM 硬检索 → Top-K → Target Attention |
| 视频ID | item_id | embedding，共享或独立于物品塔 |
| 作者ID | author_id | embedding |
| 视频多模态 | 标题文本、封面图、前 3 帧、音频特征 | 预训练模型抽成固定 256d 向量，直接输入 MLP |
| 视频统计 | 时长(分段)、历史 CTR、完播率、点赞率、发布时间 | 分桶离散化 |
| 上下文 | 请求来源（推荐流/同城/搜索） | embedding |

**出彩点：** 采用**特征动态消融与 AutoML 自动特征组合**，在训练中筛选高效特征交叉，体现工程化调优经验。

## 6. 关键工程与学术延伸（面试可谈）

- **分位数召回的向量检索适配**：可讨论如何通过多塔输出多向量构建多索引，或利用乘积量化压缩分位数向量。
- **兴趣生命周期学习**：对长期序列引入时间衰减因子，让模型感知“兴趣偏移”。
- **去偏处理**：在损失中加入倾向性加权（IPW），纠正曝光偏差对分位数标签的影响。
- **蒸馏联动**：召回模型的分位数向量可蒸馏自精排模型的分位数塔，形成端到端一致性。

这个方案既具备完整的工程落地链条，又在目标定义、负采样、多兴趣建模上体现了前沿思考，足以成为简历中的强力亮点。
