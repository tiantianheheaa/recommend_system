- 问题：
- 视频推荐场景，召回阶段，u2i双塔召回，多目标（有点赞、关注等互动类目标，有3s快滑、7s播放等时长类目标，有分位数目标）。 请设计模型结构。 并给出模型训练、在线推理emb，以及在线召回的流程。

### 1. 模型结构：MMOE 多目标双塔 + 统一召回嵌入

采用 **User Tower** 和 **Item Tower** 分离的双塔结构，塔内利用 **MMOE** 支持多目标学习，并在末端生成一个融合后的统一召回嵌入。

**User 塔**
- 输入：用户特征（user id, 历史行为序列, 画像等），通过 Embedding 层 + 特征交叉层得到固定维度向量 `h_u`。
- 共享底层：`h_u` 后接多个 **Expert 子网络**（如 3~4 个 MLP），输出 `E_u1, E_u2, ..., E_uM`。
- 任务特定门控：每个任务 `t`（共 T 个任务）学习一个 Gate 网络，对 Experts 输出做加权和，得到任务特定表示 `u_t = Σ_i g_{t,i} * E_ui`。
- 统一召回嵌入：额外设计一个 **融合 Gate**（或注意力层），对各 `u_t` 再做一次融合，得到 `u_recall`，用于后续单向量 ANN 检索。

**Item 塔**
- 完全对称：输入物品特征（video id, 作者, 分类等）→ 共享底层 → 多个 Experts → 任务特定 Gate 输出 `v_t` → 融合层输出 `v_recall`。

**多目标预测头**
- 互动类（二分类）：点赞、关注。  
  得分 `s_t = σ(u_t · v_t)`，其中 `σ` 为 sigmoid。
- 时长类：  
  - 3s 快滑（负向信号）：按二分类建模，`s_skip = σ(u_skip · v_skip)`，正样本为“播放＜3s 且滑动”。  
  - 7s 播放（正向）：二分类，`s_play7 = σ(u_play7 · v_play7)`。
- 播放时长分位数（回归）：  
  选 P50、P70、P90 三个分位数，每个分位数 τ 对应一个任务，直接预测连续值 `ŷ_τ = u_τ · v_τ`，损失用 Pinball Loss。

**统一召回得分**  
`S_recall = u_recall · v_recall`，作为全量召回的代理分数，训练时赋予一个主目标（例如 “有效播放” 正样本的二分类损失），使其能综合反映多目标偏好。

### 2. 多目标跷跷板效应应对方法

1. **MMOE 结构**  
   共享多个 Expert，每个任务通过独立 Gate 组合 Expert，不同任务可复用共同模式，又能保有个性化空间，天然缓解负迁移。

2. **动态任务权重**  
   使用 **Uncertainty Weighting**（同方差不确定性）对每个任务的 loss 进行自适应加权：
   ```math
   L_total = Σ_t (1/(2σ_t²) * L_t + log σ_t)
   ```
   或者采用 **DWA**、**GradNorm** 动态调整权重，避免某一个任务主导梯度。

3. **梯度手术**  
   在更新塔参数时，对来自不同任务头的梯度做 **PCGrad**：若两个任务梯度方向余弦相似度＜0，则将其中一个投影到另一个的正交方向，减少冲突。

4. **任务分组**  
   互动类和时长类目标可先在各组内共享部分 Expert，再在上层组合，进一步降低冲突。例如将互动类 Gate 与时长类 Gate 分别使用两组 Expert。

### 3. 多 Embedding 的在线召回效率平衡

#### 方案 A（**推荐**）：训练融合 Embedding，线上单向量检索  
- 在塔内生成 `u_recall` 和 `v_recall`，训练时增加召回损失（如用点击/有效播放作为正样本做点积 + 交叉熵），并鼓励 `u_recall·v_recall ≈ Σ_t w_t (u_t·v_t)`（可加蒸馏 loss）。  
- **在线只使用 `u_recall` 和 `v_recall`**，一次 ANN 检索即可得到兼顾多目标的候选集，完全没有多 Embedding 开销。

#### 方案 B：多 Embedding 多路检索 + 结果融合  
如果必须在线上保留多个任务嵌入（`u_t`, `v_t`），可这样平衡效率：
- **多路并行**：为每个任务 `t` 构建物品向量的独立索引，用户请求到达后，并发请求所有路，每路返回 Top-K_t（K_t 可不同，如主要目标 K 大，次要目标 K 小）。  
- **向量索引复用**：如果所有 `v_t` 维度相同，可将多个 `v_t` 拼接存入同一索引（如 Faiss 的 `IndexIDMap` 中存多条向量），用户侧也拼接 `u_t` 计算相似度；或使用 **多向量查询（Multi-Vector）** 近似算法（如通过 PQ 编码多向量）。  
- **结果归并与截断**：各路返回的 item 合并去重后，用加权融合分数 `Σ w_t (u_t·v_t)` 重新排序截断到 Top-N，控制进入精排的候选量。  
- **算力控制**：限制并发路数（选最重要的 3~4 个任务）、减少每路检索数量，总计算量可控。

**效率对比**：方案 A 最优，几乎不增加在线检索代价；方案 B 需 N 倍检索，但可并行，通过缩小 K 维持延迟。

### 4. 模型训练流程

**样本构造**
- 日志中曝光事件作为样本主体，补充后验标签：
  - `label_like`（点赞：0/1）
  - `label_follow`（关注：0/1）
  - `label_skip3s`（快滑：播放时长 < 3s 且滑走则为 1，否则 0）
  - `label_play7s`（播放 ≥ 7s 为 1，否则 0）
  - `play_duration`（真实播放秒数，>0）
- 丢失部分标签的样本（如无互动）仅计算有标签的 loss。

**损失函数**
- 分类任务：`BinaryCrossEntropy(s_t, label_t)`。
- 分位数回归任务（τ = 0.5, 0.7, 0.9）：
  ```math
  L_τ = Σ max(τ (y - ŷ_τ), (τ-1)(y - ŷ_τ))
  ```
- 统一召回损失：将 `S_recall` 视为主召回分数，用 “有效播放” （如播放 ≥ 7s 或时长 P70 以上）做正样本，随机负采样 + batch 内负样本，计算交叉熵。
- 融合蒸馏损失（可选）：`MSE(S_recall, Σ w_t s_t)`，使融合 Embedding 对齐多目标综合得分。

**优化策略**
- 采用 Uncertainty Weighting 合并多任务 loss。
- 将 User / Item 塔分开存储，便于在线服务的 Embedding 导出。
- 必要时对互动类和时长类目标使用 PCGrad。

---

### 5. 在线推理 Embedding 与召回流程（方案 A）

**离线阶段**
- 训练完成后，固定 Item 塔参数。
- 对所有候选视频推理，输出 **统一召回嵌入 `v_recall`**，构建向量索引（如 Faiss HNSW）。
- 同时可存储各任务 `v_t` 以备后续精排使用（非召回必须）。

**在线推理**
1. 用户请求到达，携带其特征。
2. User Tower 前向计算，仅执行至得到 `u_recall`（丢弃 `u_t` 以省算力）。
3. 得到用户当前时刻的统一召回向量。

**在线召回**
1. 将 `u_recall` 传入 Item 向量索引，进行 ANN 检索，返回 Top-M（如 500~2000）个候选视频。
2. 可选：对候选视频请求各任务 `u_t·v_t` 快速打分，计算加权融合得分，截断到最终召回量 N（如 200~500）。
3. 输出候选集送入精排。

---

### 总结
- **模型**：基于 MMOE 的双塔结构，输出多任务专属嵌入 + 统一召回嵌入。  
- **防跷跷板**：MMOE、动态权重、PCGrad 多管齐下。  
- **多 Embedding 效率**：推荐使用统一召回嵌入做单向量检索；若必须保留多 Emb，通过多路并行 + 小 K 值 + 结果归并控制开销。  
- **训练与部署**：联合多目标损失 + 统一召回损失训练，在线仅使用融合向量做一次 ANN 检索，兼顾效果和效率。
