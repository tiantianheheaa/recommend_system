下面用一个**具体的数据例子**，一步步演示分位数 Label 是如何生成的。

---

## 场景设定

假设我们有 3 个视频时长分桶（模拟真实短视频场景）：

| 分桶编号 | 视频时长范围 | 说明 |
|---------|------------|------|
| 桶 A | 0 ~ 15 秒 | 超短视频（如 10s 卡点视频） |
| 桶 B | 15 ~ 60 秒 | 常规短视频（如 30s 美食教程） |
| 桶 C | 60 秒以上 | 较长视频（如 120s 剧情短片） |

---

## 第一步：收集训练数据，按桶统计

从历史日志中，我们收集到每个桶内的**观看时长**样本：

**桶 A（0~15s）的观看时长样本（已按升序排列）：**
```
[2, 3, 5, 8, 12] 秒
```
共 5 个样本。

**桶 B（15~60s）的观看时长样本（已按升序排列）：**
```
[10, 20, 35, 50, 58] 秒
```
共 5 个样本。

**桶 C（60s+）的观看时长样本（已按升序排列）：**
```
[30, 60, 120, 180, 300] 秒
```
共 5 个样本。

---

## 第二步：来了一条新样本，计算其分位数

现在有一条新曝光记录：

- **视频时长**：25 秒 → 属于 **桶 B（15~60s）**
- **用户实际观看时长**：**40 秒**

### 计算经验分位数

在桶 B 的排序数组中查找 40 的位置：
```
桶 B: [10, 20, 35, 50, 58]
            ↑
          40 落在 35 和 50 之间
```

有 **3 个样本** 的观看时长 ≤ 40（分别是 10, 20, 35）。

因此，该样本在该桶内的**经验分位数值**为：

$$
\text{Quantile} = \frac{\text{小于等于该值的样本数}}{\text{桶内总样本数}} = \frac{3}{5} = 0.6
$$

---

## 第三步：生成模型可用的 Label

基于这个 0.6 的分位数值，我们生成两种 Label：

### 1. 分位数回归的连续 Label（用于 Quantile Tower）

直接作为监督信号输入模型：
```python
quantile_target = 0.6  # 连续值，范围 [0, 1]
```

模型预测的是：给定用户和视频特征，观看时长落在桶内分布的哪个相对位置。

### 2. 分桶离散 Label（可选，用于辅助任务）

假设我们设定 **5 个分位桶**（即把 [0,1] 均分为 5 段）：

| 分位区间 | 桶索引 | 含义 |
|---------|-------|------|
| 0.0 ~ 0.2 | 0 | 观看时长在桶内垫底 |
| 0.2 ~ 0.4 | 1 | 偏短 |
| 0.4 ~ 0.6 | 2 | 中等 |
| 0.6 ~ 0.8 | 3 | 偏长 |
| 0.8 ~ 1.0 | 4 | 观看时长在桶内拔尖 |

计算 bin 索引：
```python
bin_idx = min(int(0.6 * 5), 4) = min(3, 4) = 3
```

对应的 **One-Hot 标签**：
```python
quantile_bin_label = [0, 0, 0, 1, 0]
```

---

## 为什么要分桶？—— 去偏的核心逻辑

如果不做分桶，直接用原始观看时长（40秒）做回归，会发生什么问题？

| 样本 | 视频时长 | 观看时长 | 原始值大小 | 实际兴趣强度 |
|------|---------|---------|-----------|------------|
| 样本 X | 10 秒 | 8 秒 | 8 | 很高（看了 80%）|
| 样本 Y | 120 秒 | 40 秒 | 40 | 一般（看了 33%）|

只看原始时长，40 > 8，模型会认为样本 Y 的用户兴趣更强。但实际上，样本 X 看完了视频的 80%，兴趣明显更强。

**分桶后比较的是"在同类型视频中的相对表现"：**
- 样本 X（10s 视频，看 8s）→ 在桶 A 中可能处于 0.8 分位 → 兴趣很强
- 样本 Y（120s 视频，看 40s）→ 在桶 C 中可能处于 0.3 分位 → 兴趣一般

这样生成的 Label 消除了"视频越长，观看时长绝对值越大"的**混淆偏置（Confounding Bias）**，让模型学到的是**真实的用户兴趣强度**，而非视频时长的伪相关。

## 完整流程图

```
原始日志
  │
  ├── 视频时长 25s ──→ 归入桶 B (15~60s)
  │
  ├── 观看时长 40s ──→ 在桶 B 排序数组 [10,20,35,50,58] 中定位
  │                      │
  │                      ▼
  │              3个样本 ≤ 40
  │                      │
  │                      ▼
  │              分位数 = 3/5 = 0.6
  │                      │
  └──────────────────────┤
                         ▼
              ┌─────────────────────┐
              │  连续 Label: 0.6    │ ──→ 分位数回归任务 (Pinball Loss)
              │  离散 Label: [0,0,0,1,0] │ ──→ 可选辅助分类任务
              └─────────────────────┘
```

---

## 代码对应

回顾之前提供的代码，上述过程对应这两行：

```python
generator = QuantileLabelGenerator(n_bins=5, duration_edges=[0, 15, 60, 9999])
generator.fit(train_df)  # 用训练集统计各桶的 CDF

# 对新样本生成 Label
quantile_bin, quantile_val = generator.transform(
    watch_time=40,       # 用户看了40秒
    video_duration=25    # 视频时长25秒
)
# 返回: quantile_bin = [0,0,0,1,0], quantile_val = 0.6
```

## 总结

| 概念 | 说明 |
|------|------|
| **分位数 Label 是什么** | 观看时长在"同视频时长桶"内的经验累积分布位置，范围 [0,1] |
| **为什么不用原始时长** | 消除视频时长偏置，避免模型把"长视频"误判为"高兴趣" |
| **怎么生成** | 1. 按视频时长分桶 → 2. 桶内排序 → 3. 计算经验分位数 → 4. 映射为连续值或离散桶 |
| **怎么用** | **连续值**直接用于 Quantile Tower 的 Pinball Loss；**离散值**可选用于辅助分类或做特征 |
