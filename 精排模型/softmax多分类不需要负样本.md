- softmax多分类不需要显式的负样本，因为正确的类目只有一个，其他的类目都是负例。
- ctr单点预估需要正样本和负样本，因为单点预估是一个值（sigmoid后0-1之间的值），既需要告诉模型什么时候偏向1，又需要告诉模型什么时候偏向0。  除非ctr的label改成softmax二分类。

--- 
# DNN 做 Softmax 多分类：训练样本如何构造？是否需要负样本？

## 1. 核心结论

| 场景 | 是否需要负样本 | 原因 |
|---|---:|---|
| **普通 Softmax 多分类**，例如图片分类、类目分类、意图分类 | **不需要额外构造负样本** | 每条样本只需要一个真实类别标签，Softmax + Cross Entropy 会自动把其他类别当作“竞争类” |
| **类别数极大**，例如 item_id 多分类、词表预测、召回模型 | **通常需要采样负类** | 全量 Softmax 计算成本太高，常用 sampled softmax、负采样、NCE 等近似训练 |
| **只有正反馈数据**，例如只知道用户点击/购买了某商品 | **不能简单只用正样本训练概率模型** | 缺少“未点击/未购买”的对照样本，模型无法学习什么情况下不应预测为高概率 |
| **CTR 预估** | **必须有正样本和负样本** | CTR 的定义是曝光后点击概率，需要同时学习点击与未点击曝光 |

---

# 2. Softmax 多分类的训练样本怎么构造？

## 2.1 标准多分类任务

假设要做一个商品类目分类任务，有 4 个类别：

```text
0: 手机
1: 电脑
2: 家电
3: 图书
```

每条训练样本由两部分组成：

```text
x = 样本特征
y = 真实类别label
```

例如：

| 样本 | 特征 x | 标签 y |
|---|---|---|
| 样本1 | 商品标题、图片、价格等 | 手机 |
| 样本2 | 商品标题、图片、价格等 | 图书 |
| 样本3 | 商品标题、图片、价格等 | 家电 |

模型输出为：

```text
logits = [z0, z1, z2, z3]
```

经过 Softmax 后得到每个类别的概率：

```text
p0, p1, p2, p3 = softmax(logits)
```

如果样本1的真实标签是“手机”，则它的 one-hot 标签是：

```text
y = [1, 0, 0, 0]
```

交叉熵损失为：

```text
Loss = -log(p_手机)
```

也就是希望模型把“手机”的概率拉高，同时由于 Softmax 概率和为 1，其他类别的概率会被自然压低。

---

# 3. Softmax 多分类需要负样本吗？

## 3.1 普通多分类：不需要额外构造负样本

对于标准 Softmax 多分类，每条样本虽然只有一个正标签，但在训练时：

```text
真实类别 = 正类
其他类别 = 隐式负类
```

例如一个样本真实类别是“手机”：

```text
正类：手机
负类：电脑、家电、图书
```

所以不需要人为再构造：

```text
手机 vs 非手机
手机 vs 电脑
手机 vs 家电
手机 vs 图书
```

Softmax 本身已经在做多类别竞争。

---

## 3.2 但类别数很大时，可能需要负采样

如果类别数非常大，例如：

- 预测用户下一个点击的 item_id；
- 从千万商品中预测用户会买哪个商品；
- NLP 中预测下一个词；
- 召回模型中预测用户感兴趣的 item。

此时 Softmax 输出层可能有几百万、几千万个类别。

如果每次训练都计算：

```text
所有 item 的 logits
所有 item 的 softmax 概率
```

计算成本会非常高。

这时常见做法是：

| 方法 | 思路 |
|---|---|
| **Sampled Softmax** | 每次只计算真实正类 + 一批采样负类 |
| **Negative Sampling** | 对每个正样本采若干负样本，做二分类或 pairwise 学习 |
| **NCE** | 把多分类问题转成区分真实样本和噪声样本的问题 |
| **Hierarchical Softmax** | 用树结构减少计算复杂度 |

例如用户点击了 item A，可以构造：

```text
正样本：user, item A
负样本：user, item B
负样本：user, item C
负样本：user, item D
```

模型学习：

```text
score(user, item A) > score(user, item B/C/D)
```

这里的负样本不是因为 Softmax 多分类理论上必须要，而是因为 **类别空间太大，全量 Softmax 计算太贵**。

---

# 4. “只有正样本就行吗？”

要区分两种情况。

## 4.1 对于有明确类别标签的多分类任务：可以

例如商品类目分类：

```text
商品A 属于 手机
商品B 属于 图书
商品C 属于 家电
```

每条样本都是某个类别的正样本，但整体数据集中包含了所有类别的样本。

这种情况下可以说：

> **每条样本只需要一个正标签，不需要单独标注负标签。**

因为对于样本A：

```text
手机是正类
其他类目都是隐式负类
```

---

## 4.2 对于点击、购买、转化这类行为预测：不可以只用正样本

例如你只有用户点击过的样本：

```text
用户U1 点击 商品A
用户U2 点击 商品B
用户U3 点击 商品C
```

如果只用这些样本训练模型，模型只会看到“用户喜欢什么”，但看不到：

```text
用户U1 看到了商品D但没有点击
用户U2 看到了商品E但没有点击
用户U3 看到了商品F但没有点击
```

结果模型无法学习：

- 什么样的商品用户不感兴趣；
- 什么样的曝光不会被点击；
- 什么样的用户-商品组合 CTR 低；
- 曝光位置、创意、价格、类目等因素对“不点击”的影响。

所以对于 CTR/CVR/转化率预估，只有正样本是不够的。

---

# 5. DNN 做 CTR 预估，为什么需要正样本和负样本？

## 5.1 CTR 的定义决定了必须有正负样本

CTR 是：

```text
CTR = 点击次数 / 曝光次数
```

从机器学习角度，CTR 模型学习的是：

```text
P(click = 1 | user, item, context)
```

也就是在一次曝光发生的条件下，用户点击的概率。

因此训练样本应该来自 **曝光日志**：

| 样本类型 | 含义 | label |
|---|---|---:|
| 正样本 | 曝光后发生点击 | 1 |
| 负样本 | 曝光后未发生点击 | 0 |

例如：

| user_id | item_id | position | time | 是否点击 |
|---|---|---|---|---:|
| U1 | I1 | 首页首屏 | 晚上8点 | 1 |
| U1 | I2 | 首页首屏 | 晚上8点 | 0 |
| U2 | I3 | 搜索结果页 | 下午3点 | 0 |
| U3 | I4 | 推荐流 | 晚上10点 | 1 |

模型训练目标通常是二分类交叉熵：

```text
Loss = -[y * log(p) + (1 - y) * log(1 - p)]
```

其中：

```text
y = 1 表示点击
y = 0 表示未点击
p = 模型预测的点击概率
```

---

## 5.2 如果只有点击正样本，会出现什么问题？

假设训练集中只有：

```text
点击样本 label = 1
点击样本 label = 1
点击样本 label = 1
```

模型最容易学到的策略是：

```text
所有样本都预测为高 CTR
```

因为它从未见过负样本，也就不知道哪些曝光不应该点击。

这会带来几个问题：

### 1. 概率不可校准

模型输出不再是可靠的 CTR。

例如真实情况：

```text
商品A真实CTR = 5%
商品B真实CTR = 0.5%
```

但只有正样本时，模型可能都预测得很高，无法正确反映点击概率。

---

### 2. 无法区分低质流量

负样本告诉模型：

```text
这个用户看到了但没有点
这个位置曝光了但没有点
这个创意展示了但没有点
这个商品被推荐了但没有点
```

这些信息对于学习低 CTR 模式非常关键。

---

### 3. 排序目标会偏向热门内容

如果只看点击样本，热门商品、强曝光商品天然点击次数多。

模型容易把：

```text
曝光多导致点击多
```

误认为：

```text
用户真正偏好高
```

所以 CTR 训练一定要基于曝光样本，而不是只基于点击样本。

---

# 6. Softmax 多分类与 CTR 二分类的关键区别

| 对比项 | Softmax 多分类 | CTR 预估 |
|---|---|---|
| 任务目标 | 判断样本属于哪个类别 | 预测一次曝光是否会点击 |
| 标签形式 | 单一类别 label | click=1 或 click=0 |
| 输出形式 | 多个类别概率，概率和为1 | 一个点击概率 |
| 损失函数 | Softmax Cross Entropy | Binary Cross Entropy |
| 是否需要显式负样本 | 普通多分类不需要 | 需要 |
| 负类来源 | 同一个样本下其他类别天然是负类 | 曝光未点击样本 |
| 样本基础 | 有标注类别的样本 | 曝光日志 |
| 典型例子 | 类目分类、意图识别 | 推荐广告点击率预估 |

---

# 7. 一个容易混淆的例子：推荐系统里预测用户点击哪个 item

如果把推荐建模成 Softmax 多分类：

```text
输入：user/context
输出：所有 item 的点击概率分布
label：用户点击的 item_id
```

理论上每条点击样本的正类是被点击 item，其他所有 item 是负类。

但这里有一个问题：

> 用户没有点击某个 item，并不一定代表用户不喜欢它，可能只是没有曝光给用户。

所以推荐/广告里的负样本最好来自：

```text
已曝光但未点击的 item
```

而不是随便从全库随机采。

更合理的 CTR 样本是：

```text
user, item, context, exposed=1, click=0/1
```

而不是只用：

```text
user, clicked_item
```

# 8. 实践建议

## 8.1 Softmax 多分类任务

样本构造建议：

```text
features: x
label: class_id
```

要求：

- 类别标签互斥；
- 每条样本有且只有一个主标签；
- 训练集覆盖各类别；
- 类别不均衡时可做 class weight、重采样或 focal loss；
- 类别数巨大时考虑 sampled softmax 或 hierarchical softmax。

---

## 8.2 CTR 预估任务

样本构造建议：

```text
features: user特征 + item特征 + context特征 + ad特征
label: 是否点击
```

正样本：

```text
曝光后点击，label = 1
```

负样本：

```text
曝光后未点击，label = 0
```

注意事项：

- 负样本必须来自真实曝光；
- 注意点击归因窗口，例如曝光后多久内点击算正样本；
- 避免未来信息泄露；
- 训练样本分布要尽量贴近线上 serving 分布；
- 如果负样本过多，可以做负采样，但需要注意样本权重校正；
- 如果做 pCTR 校准，需要处理采样偏差。

---

# 9. 总结

**Softmax 多分类**中，每条样本只需要一个真实类别标签，其他类别在损失函数里天然充当隐式负类，所以普通多分类不需要额外构造负样本。

**CTR 预估**中，目标是预测一次曝光后的点击概率，训练样本必须来自曝光日志，既要有点击样本，也要有未点击样本。点击样本告诉模型“什么会点”，未点击样本告诉模型“什么不会点”，二者共同决定模型能否学到可靠的点击概率。

一句话概括：

> **Softmax 多分类的负类来自类别竞争；CTR 预估的负样本来自真实曝光未点击。**
